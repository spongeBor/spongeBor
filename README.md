## About me

I am a T-shaped engineer: deep in frontend architecture and broad across the full software delivery lifecycle.

- Build complex web applications with **TypeScript, React, Vue and Next.js**
- Design reusable UI architecture, state models and clear module boundaries
- Connect frontend systems to **Node.js, Fastify, Go and data services**
- Care about performance, observability, maintainability and developer experience

## Areas of depth

### Frontend architecture

- Model complex product behavior as explicit state and predictable data flow
- Keep domain logic independent from UI frameworks where practical
- Design reusable components without hiding important product semantics
- Balance rendering performance, code clarity and delivery speed

### System and API design

- Connect frontend, backend and data layers through explicit contracts
- Treat errors, asynchronous state and side effects as observable workflows
- Avoid unnecessary coupling between business rules and framework details
- Choose abstractions according to current complexity and expected change

### Engineering quality

- Make build, test, release and deployment processes repeatable
- Prefer fast feedback, visible failures and actionable diagnostics
- Measure before optimizing and understand the domain before abstracting
- Treat maintainability as a long-term product capability

## How I approach engineering

```text
Understand the domain
        ↓
Define clear boundaries
        ↓
Design explicit contracts
        ↓
Build the simplest reliable solution
        ↓
Measure, validate and iterate
```

> Good engineering is not about maximizing abstraction. It is about making complexity visible, controlled and changeable.
