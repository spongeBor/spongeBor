## 🔬 Areas of depth

### Frontend architecture

- 使用清晰的模块边界控制复杂度
- 设计可预测的状态与数据流
- 分离业务领域、UI 表达与基础设施实现
- 关注大型应用中的渲染性能和交互体验
- 通过类型系统降低跨模块协作成本

### System and API design

- 使用明确的契约连接前端、服务端与数据层
- 将错误、状态和副作用设计为可观察、可处理的流程
- 避免业务逻辑与框架细节过度耦合
- 在抽象能力与实现复杂度之间保持平衡

### Engineering quality

- 让构建、测试、发布和部署流程可重复
- 优先建立反馈快速、失败明确的开发流程
- 在优化之前测量，在抽象之前理解问题
- 将可维护性视为产品长期能力的一部分
